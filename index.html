The short story of my way as developer

01 Design.

It was the first thing I learned about IT. So before I started developing, I spent some time at figma to practice design and understand the basic principles of UI/UX. This knowledge has helped me to this day

02. Frontend. Part 1

Logically, after designing, I started to learn Frontend. JS didn't seem convenient to me, so after learning the basics and practicing layout on HTML and CSS, I decided to go further, which led me to C#.

03. Desctop.

C# seemed much more correct and understandable to me, after figuring out oop I started writing my first pet projects. Then I got acquainted with WPF and MVVM. I spent a lot of time using this bundle, but the need for commercial development made me go further again

04. Backend.

Since by that time I knew C# just fine, I chose ASP.NET as a framework almost without thinking. As the volume of data grew, I learned SQL and EF core. At first I used MVC, later, having learned about client-server architecture, I switched to REST API.

05. Cloud services.

When several APIs were ready, there was a need to host them. trusting microsoft, i chose azure as my cloud provider. today, almost all my APIs and databases are hosted in azure, you can also test them by visiting the homepages

06. Frontend. Part 2

To bring my own APIs to life, I started learning frontend again. After gaining basic knowledge, I immediately started learning React and TS. In autumn 2023, I started commercial web development using all the knowledge I had accumulated.

I've come a long way, but I'm still going.
